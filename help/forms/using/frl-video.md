# Script de la vidéo : configurer une licence avec fonctionnalités restreintes (FRL) pour Adobe Acrobat dans AEM Forms

| Visuel | Titre | Script de narration |
|--------|-------|---------------|
| Écran de titre avec logo AEM Forms et illustration FRL | Introduction | Bienvenue dans ce tutoriel sur la configuration d’une licence avec fonctionnalités restreintes (FRL) pour Adobe Acrobat sur votre serveur AEM Forms. Ce processus est essentiel, car Adobe abandonne les licences perpétuelles pour adopter le nouveau modèle FRL. |
| Chronologie indiquant les dates de fin de vie avec des flèches pointant sur la transition vers les FRL | Contexte | Les versions d’Adobe Acrobat Classic avec des licences perpétuelles arrivent en fin de vie, et nécessitent donc une transition vers des versions plus récentes. AEM Forms PDF Generator utilise Adobe Acrobat pour les fonctionnalités de conversion de documents. Le modèle de licence avec fonctionnalités restreintes offre les mêmes fonctionnalités, mais utilise une approche de licence moderne gérée via Adobe Admin Console. |
| Liste de contrôle des éléments requis avec des cases à cocher et des icônes | Prérequis | Avant de commencer, nous allons nous assurer que vous disposez de tout ce dont vous avez besoin : premièrement, un accès à Adobe Admin Console avec les privilèges d’administration système. Deuxièmement, un compte d’utilisation doté du rôle Administration de déploiement dans Admin Console. Troisièmement, des privilèges d’administration locale sur la machine exécutant AEM Forms. Quatrièmement, un système d’exploitation Windows 64 bits sur votre serveur AEM Forms. Cinquièmement, un connexion Internet stable pour l’activation des licences. Vous devez également vous familiariser avec l’interface d’Adobe Admin Console et connaître votre architecture de déploiement AEM Forms. |
| Icône d’avertissement avec image d’illustration | Remarque importante | N’oubliez pas de sauvegarder vos paramètres Acrobat personnalisés avant de commencer, car vous devrez désinstaller les installations Acrobat existantes au cours de ce processus. |
| Organigramme présentant les étapes numérotées de 1 à 7 | Vue d’ensemble du processus | Ce processus comprend plusieurs étapes importantes : la préparation d’un package FRL dans Admin Console, l’octroi d’autorisations de téléchargement, la désinstallation des versions précédentes d’Acrobat, l’installationde la dernière version d’Acrobat Pro, le déploiement du package FRL et la vérification de l’installation. Passons en revue chaque étape. |
| Capture d’écran de l’interface d’Adobe Admin Console avec l’onglet Packages mis en surbrillance | Étape 1 : préparation d’Admin Console | Tout d’abord, connectez-vous à Adobe Admin Console avec les privilèges d’administration système. Accédez à l’onglet Packages et sélectionnez « Licence avec fonctionnalités restreintes ». Cliquez sur le bouton « Commencer » pour commencer à créer votre package. |
| Écran Créer une configuration de package avec les paramètres mis en surbrillance | Configuration du package | Configurez votre package avec les paramètres recommandés suivants : sélectionnez « Hors ligne » pour la méthode d’activation, « Génération de PDF » pour les droits et « Windows 64 bits » pour la plateforme. Pour la sélection des applications, conservez uniquement le fichier de licence dans les applications sélectionnées. Donnez à votre package un nom explicite tel que « Acrobat FRL AEM Forms » et créez le package. |
| Onglet Utilisateurs d’Admin Console avec la boîte de dialogue des autorisations | Étape 2 : octroyer des autorisations de téléchargement | Ensuite, vous devez créer ou identifier un utilisateur ou une utilisatrice qui téléchargera le package. Dans Admin Console, accédez à l’onglet Utilisateurs, sélectionnez votre utilisateur ou votre utilisatrice et attribuez-lui le rôle « Administrateur de déploiement ». Cela lui permet de télécharger le package FRL que vous avez créé. |
| Panneau de configuration Windows avec la section Programmes et fonctionnalités qui affiche Acrobat | Étape 3 : désinstaller la version précédente d’Acrobat | Avant d’installer la nouvelle version, vous devez supprimer complètement les installations Acrobat existantes. Ouvrez le Panneau de configuration Windows, accédez aux Programmes, recherchez Adobe Acrobat et sélectionnez Désinstaller. Pour une suppression complète, utilisez l’outil de nettoyage Adobe Acrobat, en particulier si vous rencontrez des problèmes plus tard dans le processus. |
| Page des téléchargements d’Adobe Acrobat DC avec la section du programme d’installation mise en surbrillance | Étape 4 : télécharger et installer Adobe Acrobat Pro | Une fois la désinstallation terminée, téléchargez le programme d’installation d’Acrobat Pro approprié à partir de la page des téléchargements d’Adobe Acrobat DC. Pour la compatibilité avec PDF Generator, le programme d’installation Windows 32 bits est recommandé. |
| Processus d’installation présentant les étapes d’extraction et de configuration | Processus d’installation | Extrayez le fichier zip téléchargé dans un dossier, recherchez le fichier Setup.exe et exécutez-le en tant qu’administrateur. Suivez les instructions à l’écran pour terminer l’installation. Une fois l’installation terminée, ouvrez Acrobat Pro une fois pour ignorer les boîtes de dialogue de bienvenue et terminer la configuration initiale. |
| Onglet Packages d’Admin Console avec le bouton Téléchargement en surbrillance | Étape 5 : télécharger le package FRL | À présent, connectez-vous à Adobe Admin Console à l’aide du compte auquel vous avez précédemment accordé les autorisations de téléchargement. Accédez à l’onglet Packages, recherchez votre package FRL et téléchargez-le sur votre serveur AEM Forms. |
| Invite de commandes exécutée en tant qu’administrateur avec des commandes en cours de saisie | Étape 6 : déployer le package | Extrayez le package téléchargé dans un répertoire de votre serveur. Ouvrez l’invite de commandes en tant qu’administrateur et accédez au répertoire d’extraction. |
| Ligne de commande affichant la commande d’activation avec la syntaxe mise en surbrillance | Commande d’activation | Exécutez la commande d’activation en utilisant la syntaxe suivante : `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Remplacez le nom du fichier JSON par le nom de fichier exact de votre package. Les paramètres de commandes sont les suivants : -p pour la plateforme, -i pour installer la licence et -f pour spécifier le chemin d’accès du fichier JSON. Une fois l’opération réussie, le message « Opération terminée avec succès » s’affiche. |
| Tableau présentant les paramètres de commandes et leurs descriptions | Paramètres de commandes | Le paramètre -p spécifie la plateforme et détecte automatiquement le système d’exploitation. Le paramètre -i indique à l’outil d’installer et d’activer la licence. Enfin, le paramètre -f spécifie le chemin d’accès du fichier de licence JSON. |
| Interface d’administration d’AEM Forms affichant le service PDF Generator | Étape 7 : test et vérification | Enfin, testez votre installation en ouvrant l’interface d’administration d’AEM Forms et en convertissant un document simple en PDF à l’aide du service PDF Generator. Vérifiez également l’installation d’Acrobat en ouvrant Acrobat Pro, puis en accédant à Aide → À propos et en confirmant le numéro de version et le statut de la licence. |
| Écran partagé : boîte de dialogue À propos d’Acrobat et conversion en PDF réussie | Vérification | Vérifiez que la licence s’affiche comme activée dans Acrobat et que PDF Generator peut convertir correctement des documents. |
| Liste des erreurs courantes et leurs solutions avec des icônes | Conseils de dépannage | Si vous rencontrez des problèmes, vérifiez les points suivants : vérifiez que l’invite de commandes est bien exécutée en tant qu’administrateur, que le nom du fichier JSON est correct, si d’autres installations sont en conflit et si votre serveur dispose d’une connectivité Internet appropriée pour l’activation. |
| Éléments supplémentaires de résolution des problèmes avec des coches | Options supplémentaires de résolution des problèmes | Vérifiez également s’il existe des restrictions de pare-feu, des packages corrompus ou des identifiants Adobe en conflit sur le serveur. Si toutes les options de résolution des problèmes échouent, contactez l’assistance Adobe en précisant les messages d’erreur détaillés. |
| Écran d’opération terminée avec succès avec une icône en forme de coche et des informations de contact | Conclusion | Vous avez maintenant terminé de configurer la licence avec fonctionnalités restreintes pour Adobe Acrobat sur votre serveur AEM Forms. Si vous avez besoin d’aide supplémentaire, contactez l’assistance Adobe ou consultez la documentation détaillée disponible sur Experience League. |
| Écran de fin avec le logo Adobe et les crédits | Merci | Merci d’avoir regardé ce tutoriel. Pour plus d’informations, consultez la documentation Experience League. |
