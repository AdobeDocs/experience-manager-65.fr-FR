# Script vidéo : configuration de la gestion des licences avec restrictions de fonctionnalités (FRL) pour Adobe Acrobat sur AEM Forms

| Visuel | Titre | Script du présentateur |
|--------|-------|---------------|
| Écran de titre avec logo AEM Forms et graphique FRL. | Présentation | Bienvenue dans ce tutoriel sur la configuration de la licence restreinte de fonctionnalité (FRL) pour Adobe Acrobat sur votre serveur AEM Forms. Ce processus est essentiel, car Adobe abandonne les licences perpétuelles pour adopter le nouveau modèle de FRL. |
| Chronologie indiquant les dates de fin de vie avec des flèches pointant vers la transition FRL. | Contexte | Les versions d’Adobe Acrobat Classic avec licences perpétuelles arrivent en fin de vie, ce qui nécessite une transition vers des versions plus récentes. AEM Forms PDF Generator s’appuie sur Adobe Acrobat pour les fonctionnalités de conversion de documents. Le modèle de licence restreinte de fonctionnalités offre les mêmes fonctionnalités, mais avec une approche de licence moderne gérée via Adobe Admin Console. |
| Liste de contrôle des éléments requis avec cases à cocher et icônes | Prérequis | Avant de commencer, nous allons nous assurer que vous disposez de tout ce dont vous avez besoin : tout d’abord, l’accès à Adobe Admin Console avec les privilèges d’administrateur système. Ensuite, un compte utilisateur doté du rôle Administrateur de déploiement dans Admin Console. Troisièmement, les privilèges d’administrateur local sur le serveur exécutant AEM Forms. Quatrièmement, un système d’exploitation Windows 64 bits sur votre serveur AEM Forms. Cinquièmement, une connexion Internet stable pour l&#39;activation des licences. Vous devez également vous familiariser avec l’interface de Adobe Admin Console et l’architecture de déploiement de votre AEM Forms. |
| Icône d’avertissement avec graphique de sauvegarde | Remarque importante | N’oubliez pas de sauvegarder les paramètres Acrobat personnalisés avant de commencer, car vous devrez désinstaller les installations Acrobat existantes au cours de ce processus. |
| Diagramme de flux présentant les étapes numérotées de 1 à 7. | Présentation du processus | Ce processus implique plusieurs étapes importantes : la préparation d&#39;un package FRL dans Admin Console, l&#39;octroi d&#39;autorisations de téléchargement, la désinstallation des versions précédentes d&#39;Acrobat, l&#39;installation de la nouvelle Acrobat Pro, le déploiement du package FRL et la vérification de l&#39;installation. Passons en revue chaque étape. |
| Capture d’écran de l’interface de Adobe Admin Console avec l’onglet Packages mis en surbrillance | Étape 1 : Préparation D’Admin Console | Tout d’abord, connectez-vous à Adobe Admin Console avec les privilèges d’administrateur système. Accédez à l’onglet Packages et sélectionnez « Licence restreinte pour les fonctionnalités ». Cliquez sur le bouton « Commencer » pour commencer à créer votre package. |
| Écran Créer une configuration de package avec les paramètres mis en surbrillance | Configuration du package | Configurez votre package avec les paramètres recommandés suivants : sélectionnez Hors ligne pour la méthode d’activation, Génération PDF pour les droits et Windows 64 bits pour la plateforme. Pour la sélection d&#39;applications, conservez uniquement le fichier de licence dans les applications sélectionnées. Donnez à votre package un nom explicite tel que « Acrobat FRL AEM Forms » et créez le package. |
| Onglet Utilisateurs d’Admin Console avec boîte de dialogue des autorisations | Étape 2 : Octroi Des Autorisations De Téléchargement | Ensuite, vous devez créer ou identifier un utilisateur qui téléchargera le package. Dans Admin Console, accédez à l’onglet Utilisateurs , sélectionnez votre utilisateur et attribuez-lui le rôle « Administrateur de déploiement ». Cela leur permet de télécharger le package FRL que vous avez créé. |
| Panneau de Contrôle Windows avec applications et fonctionnalités présentant Acrobat | Étape 3 : désinstaller l’Acrobat précédente | Avant d’installer la nouvelle version, vous devez supprimer complètement les installations Acrobat existantes. Ouvrez le Panneau de Contrôle Windows, accédez à Applications, recherchez Adobe Acrobat et sélectionnez Désinstaller. Pour une suppression complète, pensez à utiliser l’outil de nettoyage Adobe Acrobat, en particulier si vous rencontrez des problèmes plus tard dans le processus. |
| Page des téléchargements Adobe Acrobat DC avec la section du programme d’installation mise en surbrillance | Étape 4 : Télécharger et installer Adobe Acrobat Pro | Après la désinstallation, téléchargez le programme d’installation d’Acrobat Pro approprié à partir de la page des téléchargements d’Adobe Acrobat DC. Pour la compatibilité avec PDF Generator, le programme d’installation de Windows 32 bits est recommandé. |
| Processus d’installation présentant les étapes d’extraction et de configuration | Processus d’installation | Extrayez le fichier zip téléchargé dans un dossier, recherchez Setup.exe et exécutez-le en tant qu’administrateur. Suivez les instructions à l’écran pour terminer l’installation. Après l’installation, ouvrez Acrobat Pro une fois pour ignorer les boîtes de dialogue de bienvenue et terminer la configuration initiale. |
| Onglet Packages Admin Console avec le bouton de téléchargement en surbrillance | Étape 5 : télécharger le package FRL | Désormais, connectez-vous au Adobe Admin Console à l’aide du compte auquel vous avez accordé précédemment des autorisations de téléchargement. Accédez à l’onglet Packages , localisez votre package FRL et téléchargez-le sur votre serveur AEM Forms. |
| Invite de commandes en tant qu’administrateur avec commandes saisies | Étape 6 : déployer le package | Extrayez le package téléchargé dans un répertoire de votre serveur . Ouvrez l’invite de commandes en tant qu’administrateur et accédez au répertoire d’extraction. |
| Ligne de commande affichant la commande d’activation avec la syntaxe mise en surbrillance | Commande d&#39;activation | Exécutez la commande d’activation en utilisant la syntaxe suivante : `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`. Remplacez le nom de fichier JSON par le nom de fichier exact de votre package. Les paramètres de commande sont les suivants : -p pour la plateforme, -i pour installer la licence et -f pour spécifier le chemin d’accès au fichier JSON. Une fois l’opération réussie, le message « Opération terminée avec succès » s’affiche. |
| Tableau présentant les paramètres de commande et leur description | Paramètres de la commande | Le paramètre -p spécifie la plateforme et détecte automatiquement le système d&#39;exploitation. Le paramètre -i indique à l&#39;outil d&#39;installer et d&#39;activer la licence. Et le paramètre -f spécifie le chemin d’accès au fichier de licence JSON. |
| Interface d’administration d’AEM Forms présentant le service PDF Generator | Étape 7 : test et vérification | Enfin, testez votre installation en ouvrant l’interface d’administration d’AEM Forms et en convertissant un document simple en PDF à l’aide du service PDF Generator. Vérifiez également l’installation d’Acrobat en ouvrant Acrobat Pro, en accédant à Aide → À propos et en confirmant le numéro de version et le statut de licence. |
| Écran partagé : boîte de dialogue À propos d’Acrobat et conversion réussie de PDF | Vérification | Vérifiez que la licence s’affiche comme activée dans Acrobat et que PDF Generator peut convertir correctement les documents. |
| Liste des erreurs et solutions courantes avec icônes | Conseils de dépannage | Si vous rencontrez des problèmes, vérifiez les points suivants : assurez-vous que vous exécutez l’invite de commandes en tant qu’administrateur, vérifiez que le nom de fichier JSON est correct, recherchez les installations en conflit et vérifiez que votre serveur dispose d’une connectivité Internet appropriée pour l’activation. |
| Autres éléments de dépannage comportant des coches | Plus de dépannage | Recherchez également les restrictions de pare-feu, les packages corrompus ou les identifiants Adobe en conflit sur le serveur. Si tout le reste échoue, contactez l’assistance Adobe avec des messages d’erreur détaillés. |
| Écran de réussite avec icône en forme de coche et coordonnées | Conclusion | Vous avez maintenant configuré avec succès les licences restreintes de fonctionnalités pour Adobe Acrobat sur votre serveur AEM Forms. Si vous avez besoin d’aide supplémentaire, contactez l’assistance Adobe ou consultez la documentation détaillée disponible sur Experience League. |
| Fermeture de l’écran avec le logo et les crédits Adobe | Merci | Merci d’avoir regardé ce tutoriel. Pour plus d’informations, consultez la documentation d’Experience League. |
